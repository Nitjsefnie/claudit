Activity Heatmap panel — hours × weekdays in Czech local time

Context claudit visualises Claude Code session transcripts from R2/Postgres. The user requested a new dashboard panel: an activity heatmap distributed as hours × weekdays, using the Czech timezone with correct summer-time handling. This spec decides where the timezone-aware bucketing happens and how the panel is built.

Decision
Group weekday × hour in SQL with AT TIME ZONE 'Europe/Prague'; ship a self-fetching backend-gated panel.
Postgres tzdata knows the exact CET↔CEST transition instants, so summer-time correctness is inherited rather than reimplemented. The panel follows the established ToolUsagePanel contract (per-panel model filter, SSE-driven refetch, backendOn gate).
Deliberately not done: client-side bucketing from dashboard events — those are adaptive buckets up to 1 day wide (ts = bucket center), which destroys hour-of-day information at long ranges.
01

Backend — GET /api/activity-heatmap

One new read endpoint in backend/api.py, mirroring tool_usage for params, auth, validation, and caching.

Query params range (default 30d), project, model; @cache_response; 400 on bad range via _parse_range. Module constant HEATMAP_TZ = "Europe/Prague" passed as a bound SQL parameter, never interpolated. Cross-file uuid dedup at read time (SV-PARSER-SPEC) reuses the dashboard endpoint's DISTINCT ON (r.uuid) … UNION ALL … uuid IS NULL body, with ts >= since, optional f.project_id = %s and r.model LIKE %s filters.

SELECT EXTRACT(ISODOW FROM (d.ts AT TIME ZONE %s))::int AS dow,   -- 1=Mon … 7=Sun
       EXTRACT(HOUR   FROM (d.ts AT TIME ZONE %s))::int AS hour,  -- 0 … 23
       COUNT(*)             AS requests,
       SUM(d.output_tokens) AS output_tokens,
       SUM(d.cost_usd)      AS cost_usd
FROM deduped d
WHERE d.ts IS NOT NULL
GROUP BY 1, 2

Response is sparse (only non-empty cells):

{"range": "30d", "tz": "Europe/Prague",
 "cells": [{"dow": 1, "hour": 9, "requests": 42,
            "output_tokens": 12345, "cost_usd": 1.23}, …]}
02

DST semantics — the load-bearing bit

Rejected: fixed-offset / client-side math
  • UTC+1/+2 constants reimplement DST transition rules by hand and rot silently.
  • Dashboard events lose hour-of-day fidelity at long ranges (1-day buckets).
Chosen: Postgres tzdata
  • timestamptz AT TIME ZONE 'Europe/Prague' → CET (UTC+1) winter, CEST (UTC+2) summer, transitions at the true instants.
  • Spring-forward day: local hour 02 doesn't exist → gets no events. Correct.
  • Fall-back day: local hour 02 happens twice → receives two UTC hours. Correct.
03

Frontend — ActivityHeatmapPanel

SVG 7×24 grid in src/dashboard-charts-extra.jsx, mounted in Dashboard behind the backendOn gate.

Contract: props {models, project, range, nonce} — identical to ToolUsagePanel, including the per-panel model select and refetch on [project, range, activeModel, nonce] (SSE ingest_done bumps nonce → live updates for free).

Grid: rows Mon→Sun top-to-bottom (Monday-first, Czech week convention), columns 0–23; ~2px cell gaps, rounded cells, ResizeObserver width.

Color: sequential single-hue ramp, theme surface at 0 → one accent at max; intensity sqrt(v / max) (activity is heavy-tailed). Zero cells stay at surface with a faint border. Labels wear theme text colors, never the accent.

Interaction: per-cell hover tooltip via window.DashTooltip — Tue 14:00–15:00 · 123 requests · 45.6k output tok · $1.23 (all three metrics, regardless of toggle). Metric toggle: Requests / Output tokens / Cost (default). Min→max gradient legend with the max labeled; header notes Europe/Prague (CET/CEST). Σ margin column/row carry per-weekday and per-hour totals, independently scaled; the corner cell holds the grand total.

05

Verification

How the implementer confirms correctness — concrete, runnable.

Shape. On app_with_data (r2_mini), ?range=3650d → 200, tz == "Europe/Prague", all cells dow ∈ 1..7, hour ∈ 0..23.
Dedup consistency. Σ cells[].requests == Σ dashboard hourly requests for the same range.
DST winter. Synthetic record at 2026-01-15T10:30:00Z (Thu, CET) lands in dow=4, hour=11.
DST summer. Synthetic record at 2026-07-15T10:30:00Z (Wed, CEST) lands in dow=3, hour=12.
Filters + errors. model=/project= subset the cells; bad range → 400. Full suite: python3 -m pytest tests/ -q green.
06

Out of scope

Explicitly deferred — considered, not missed.

Offline (drag-drop) heatmap
Consistent with the three newest panels (Tool Usage, Reply Latency, Tool Error Rate), all backend-only. Offline per-record events retain full fidelity, so this can be added later without redesign.
Configurable timezone
YAGNI — the operator is in CZ. HEATMAP_TZ is one constant, trivially parameterizable later.
/api/export PNG parity
The matplotlib export script doesn't know this panel; separate effort if ever wanted.
